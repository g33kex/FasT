Ce document est une fiche de dépannage expliquant la méthode d'installation de FasT, ainsi que la marche à suivre en cas de dysfonctionnement.

Dépannage



  1. Installation


1. Pré requis


A. Choix du matériel


FasT est un logiciel nécessitant beaucoup de ressources processeur, il est donc conseillé de l'exécuter sur un ordinateur possédant au moins un processeur Intel core i5, et équipé d'un minimum de 2 Go de RAM.

Aussi, vous aurez besoin d'un clavier et d'une souris à deux boutons disposant d'une molette (ou un pavé tactile supportant les mouvements de translation verticale).


B. Choix du système d'exploitation


FasT est conçut pour fonctionner sur GNU/Linux, Windows et OS X (une utilisation sous Solaris ou FreeBSD est également possible mais nécessite l'inclusion manuelle des librairies OpenGL adéquates). Cependant, des problèmes supplémentaires peuvent être rencontrés lorsqu'il est utilisé sous Windows.

FasT est optimisé pour le plein écran sous OS X avec la librairie cocoa.


C. Logiciels requis


FasT est entièrement écrit en Java, il nécessite donc une machine virtuelle Java (JVM). Assurez vous de disposer d'une version de Java supérieure à la version 1.8, que vous pouvez télécharger sur le site https://java.com/fr/. (Java appartient à Oracle et n'est en aucun cas affilié avec FasT).


2. Exécution


Vous trouverez le logiciel FasT sur la clef USB qui vous a été remise avec cette notice. Pour le lancer, il vous suffit de double cliquer sur le fichier « FasT.jar » (ou « FasT.exe » sous windows si les restrictions sur votre système vous permettent d'exécuter des fichiers exécutables). En cas de problème avec l'exécution de FasT, merci de contacter votre administrateur système.


  1. Problèmes connus


FasT étant un logiciel développé par une seule personne et étant en version bêta, il comporte un certain nombres de bugs. En voici une liste non exhaustive :

#1 : Le curseur de la souris n'est pas toujours le bon

#2 : Dans certains cas, les balles peuvent traverser les obstacles

#3 : Problèmes graphiques lors du redimensionnement

#4 : Les murs n'ont pas d'épaisseur

#5 : Les balles traversent les obstacles pendant leur déplacement par l'utilisateur

#6 : Le curseur n'est pas bloqué lors d'une tentative pour redimensionner une boite à une taille plus petite que la valeur minimum autorisée

#7 : Il y a des erreurs d'affichage des menus sous Windows

#8 : Lors de l'appui simultané de certaines touches, l'écran peut devenir rose
